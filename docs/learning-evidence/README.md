실습·검증 기록은 정답만 정리한 요약문이 아니다. 학습자가 처음 무엇을 이해했고, 실제로 무엇을 실행했으며, 예상과 결과가 어떻게 달랐고, 어떤 근거로 관점을 고쳤는지 보여 주는 기록이다.

## 적용 시점

다음 중 하나에 해당하는 문서를 작성하거나 수정하기 전에 이 규칙을 반드시 읽는다.

- 학습지나 개념 게이트에 직접 답한 내용을 저장할 때
- 초기 가설과 AI 또는 공식 자료의 피드백을 비교할 때
- 독립 실험의 예상과 실제 결과를 기록할 때
- 구현 과정에서 이해가 바뀐 이유를 학습 기록으로 남길 때

이 규칙은 Wiki의 학습 문서에만 적용한다. 실제 프로젝트의 작업 상태, 완료율, 이슈 ID를 이 디렉터리에서 추적하지 않는다.

## 반드시 보존할 정보

실습·검증 기록에는 다음 세 층이 구분되어야 한다.

1. **학습자 원문:** 질문과 답변을 맞춤법, 표현, 의문부호까지 수정하지 않고 그대로 기록한다.
2. **피드백과 정정:** AI 답변이나 참고 자료를 대화 순서대로 복사하지 않고, 오류·누락·정정 근거별로 구조화한다.
3. **현재의 판단:** 무엇을 기억해야 하는지, 다음에 어떤 질문이나 실험으로 확인할지를 명시한다.

학습자의 원문을 더 매끄러운 표현으로 바꿔 학습자 답변처럼 기록하지 않는다. 수정된 모범 답안이 필요하면 `AI 정리` 또는 `현재의 정리`라고 표시해 원문과 분리한다. 같은 질문에 다시 답했다면 이전 답을 덮어쓰지 않고 날짜와 순서를 붙여 관점 변화를 남긴다.

## 문서 구조

실습·검증 기록은 가능한 한 다음 구조를 따른다.

1. **학습 맥락:** 연결된 로드맵, 질문을 받은 이유, 증거 수준을 적는다.
2. **학습자 원문:** 직접 작성한 질문과 답변을 그대로 인용한다.
3. **판정:** 정확한 부분, 부분적으로 맞는 부분, 잘못 이해한 부분을 구분한다.
4. **놓친 부분:** 단순한 정답보다 왜 기존 설명이 불완전했는지를 자세히 설명한다.
5. **기억해야 할 기준:** 다음 설계나 실험에서 재사용할 짧은 판단 기준을 남긴다.
6. **다음 검증:** 아직 구현이나 실험으로 확인하지 않은 내용을 질문 형태로 남긴다.

증거 수준은 다음 중 하나로 명시한다.

- **개념 설명:** 자신의 말로 설명하고 피드백을 받은 상태
- **독립 실험:** 작은 코드와 실제 출력으로 확인한 상태
- **프로젝트 적용:** 관련 프로젝트의 코드·테스트·커밋으로 연결된 상태

상위 수준의 증거가 없는데 있는 것처럼 표현하지 않는다. 예시 스키마와 의사코드는 프로젝트 구현 증거가 아니다.

## 작성 원칙

- **원문과 해석 분리:** 학습자 원문, AI 피드백, 공식 자료에서 확인한 사실을 서로 다른 절에 둔다.
- **채팅 전문 금지:** AI 답변은 핵심 경계와 판단 기준을 중심으로 압축한다. 반복된 설명과 대화형 표현은 제거한다.
- **누락 중심:** 이미 정확히 이해한 내용을 길게 반복하지 않고, 놓쳤거나 혼동한 부분을 가장 자세히 쓴다.
- **정정 이유 보존:** 무엇이 틀렸는지만 적지 않고, 어떤 데이터 경계·실패 사례·반례 때문에 정정했는지 설명한다.
- **사실성 표시:** 공식 사실, 설계 해석, 추천, 미검증 가설을 `AGENTS.md`의 표기 원칙에 따라 구분한다.
- **민감정보 제외:** 질문 원문에 계정, 토큰, 개인정보, 비공개 운영 데이터가 있으면 그대로 게시하지 말고 사용자 확인을 받는다.
- **현재성 제한:** 법령·가격·제품 기능처럼 바뀔 수 있는 내용은 확인 날짜와 출처를 남긴다. 사용자가 제공한 예시는 최신 원문으로 검증했는지 구분한다.

## 파일과 연결 규칙

- 파일명은 `YYYY-MM-DD-주제.md` 형식을 사용한다.
- 같은 세션과 같은 개념 게이트의 기록은 새 파일로 나누지 않고 기존 문서에 이어 쓴다.
- 학습의 출발점이 된 로드맵이나 문서에 상대 링크를 건다.
- 독자가 찾아야 하는 대표 실습·검증 기록만 루트 `README.md`에서 연결한다.
- 실제 프로젝트 적용을 주장할 때는 관련 저장소의 파일, 테스트 또는 커밋 링크를 남긴다.

- [ ] 로드맵과 내부 링크가 실제 경로를 가리키는가?
